Vocab

  • The Internet
    • A global network of shared information that anyone with access can see.
  • HTTP
    • Stands for HyperText Transfer Protocol. It gives a website its most basic outline and shows information on a page.
  • Client
    • Hardware or software device that needs access to features that a server has.
  • Server
    • A computer program or device that gives another computer or device access so the user can use it.
  • Web Browser
    • Software that gives the user access to the world wide web or a local website.
  • Website
    • A collection of web pages in a similar or same domain name and was published by one server.
  • Web Page
    • A hypertext document on the world wide web and that is delivered by a web server to the user to be viewed or used.
  • Javascript
    • A programming language that creates interactive features on a website.
  • HTML
    • A standardized programming language that creates the text files, colors, pictures, format, etc of a website.
  • CSS
    • A programming language that creates the style of the page that originally uses HTML or XML.
  • URL
    • The name or address of a web page that is used when people search it up to access it.
  • IDE/text editor
    • A tool that can help create, build, edit, or debug a code.
  • Command line/terminal
    • A way for the user to speak directly to the computer without any other applications in the way.
  • BASH
    • Command line interpreter for computers. It can be used to manipulate files and can have a similar use to command line or terminal.
  • Operating System
    • The system in a computer that manages all of the other applications on the computer.
  • Algorithm
    • The procedure/process to solve a program or to run something.
  • Frontend vs. Backend Development
    • The frontend is in charge of doing whatever it is that the user sees when the web page or anything loads. The backend is in charge of making sure the servers run and everything else runs so the page loads smoothly and does what it is supposed to.
  • Git vs Github
    • Git is a system where you can store, manage, and keep track of your source code history and Github is a cloud where you can store what you had in your Git so you can share and access your data in places other than your main computer.
  • Programming vs Coding
    • Coding is writing the code that the machine can process and understand. Programming is creating the programs that will help the machines proccess the code.
  • Push
    • Essentially add/uplaod something to Github
  • Pull
    • Pulling something in Github tells the other person that you have made a change to what you've pulled from a branch.
  • Fork
    • When you copy something from a repository and make changes to it without it affecting the original.
  • Clone
    • When you copy a repository and download it to your computer or device.
  • repository
    • Essentially a folder that contains the files of a project.
